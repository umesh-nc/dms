Good middle ground
Banking data security — PCI / SOC 2
Check images contain routing and account numbers. Storage must be encrypted at rest and in transit. Recommend auto-deleting raw images after OCR extraction (retain structured data only). Consult your SOC 2 scope before storing check images longer than necessary.
OCR confidence and human review
No OCR is 100% accurate. Any field below a configurable threshold (recommend 85%) must be left blank and flagged amber — never auto-filled with a low-confidence guess. The accuracy engine still runs on OCR-populated rows, so all outlier and duplicate checks apply normally.
Donor matching from handwritten names
Printed names on personal checks are often informal — "Jim O'Brien" vs "James Patrick O'Brien". Fuzzy matching against the donor database with a confidence score and a disambiguation UI handles most cases. New donors with no match should flow into a "new donor" queue, not fail silently.
Memo line as fund classifier
Many donors write "CMA" or "Cathedral" in the memo field. A simple keyword-to-fund mapping table (configurable per diocese) can auto-suggest the correct fund for 60–70% of checks. For ambiguous or blank memos, default to the batch default fund and flag for review.
Recommended OCR vendor
AWS Textract has a dedicated "Analyze Check" API handling MICR, CAR/LAR, and structured check fields out of the box. Azure Form Recognizer has a pre-built check model. Both return field-level confidence scores. Estimated cost: $0.01–0.015 per check at scale.
No app download needed — PWA
The mobile capture page should be a progressive web app (PWA). Works in any iOS or Android browser, uses the native camera API, and requires no App Store distribution. The QR token handles authentication. This is critical for adoption — part-time clerks and volunteers won't install an app.
Batch default inheritance
The QR code carries the batch's default fund, GL code, gift type, and date. Every scanned check inherits these unless overridden — a CMA mail batch of 40 checks can be scanned in ~10 minutes with the clerk only confirming amounts, not retyping fund and GL for every row.
